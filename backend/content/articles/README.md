# Как редактировать статьи базы знаний

Каждая статья — один `.md`-файл в этой папке. Правки видны в приложении сразу
после сохранения в GitHub (бэкенд читает файлы заново на каждый запрос,
редеплой не нужен).

## Заголовок статьи (frontmatter)

Каждый файл начинается с блока между двумя строками `---`:

```
---
id: registration
category: start
category_label: С чего начать
title: Регистрация ИП
icon: ti-id-badge
short: Онлайн, бесплатно, 1–3 дня
---
```

- `id` — должен совпадать с именем файла (без `.md`), латиницей, только
  строчные буквы, цифры, `-` и `_`.
- `category` — одно из: `start`, `taxes`, `tools`.
- `category_label`, `title`, `short` — как показывается в приложении.
- `icon` — имя иконки из [Tabler Icons](https://tabler.io/icons), с префиксом `ti-`.

## Текст

Обычный абзац — просто текст, отделённый пустой строкой сверху и снизу.

## Плашка-предупреждение (жёлтая)

```
::warning
Уведомление о переходе на УСН подавайте в течение 30 дней.
::
```

## Плашка-совет (зелёная)

```
::tip
Совмещайте УСН + патент.
::
```

## Список ошибок (красный)

```
::danger
- Пропустить уведомление об УСН.
- Превысить лимит патента.
::
```

## Календарь дат

```
::events
25 апреля | Декларация УСН за прошлый год | УСН
28 апреля | Авансовый платёж за 1 квартал | УСН
::
```

Формат строки: `дата | что нужно сделать | для какого режима`.

## Шаги (нумерованный список)

```
1. **Через банк (рекомендуем)** — Тинькофф, Точка, Сбер сразу открывают счёт.
2. **Через nalog.gov.ru** — нужна подтверждённая запись Госуслуг.
```

Жирным — короткий заголовок шага, после тире/двоеточия — пояснение.

## Таблица

Обычная Markdown-таблица:

```
| | Самозанятый | ИП УСН 6% |
| --- | --- | --- |
| Лимит | 2,4 млн/год | до 450 млн |
```

## Добавить новую статью

Создайте новый `.md`-файл с таким же frontmatter — он появится в списке
автоматически, ничего больше менять не нужно.
